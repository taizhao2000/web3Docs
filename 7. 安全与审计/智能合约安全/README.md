## 安全设计三大红线

1. **零私钥信任机制**：任何涉及管理员提现、改费率或紧急暂停的特权接口，其 `owner` 绝不能是普通的单私钥钱包，必须在生产环境中移交至多签钱包（如 Gnosis Safe 3/5）。
2. **延迟生效原则**：重大控制权限的操作在多签批准后不能立即执行，必须通过时间锁（Timelock）合约挂起公示 48 小时以上，给社区和用户留足撤资和审计观察窗口。
3. **紧急一键熔断**：合约内必须加装对存款、借款方向的 Pausable 应急开关。一旦遭受未知的零日漏洞攻击，平台必须能够瞬间中止入金和借出，将损失锁死在当前。
