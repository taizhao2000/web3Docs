# 自建节点

## 概述

对于有特殊需求的项目，可以选择自行部署和维护区块链节点。

## 常用客户端

| 客户端 | 语言 | 用途 |
|--------|------|------|
| Geth (Go-Ethereum) | Go | 执行层客户端 |
| Nethermind | C# | 执行层客户端 |
| Besu | Java | 执行层客户端（企业友好） |
| Prysm | Go | 共识层客户端 |
| Lighthouse | Rust | 共识层客户端 |
| Teku | Java | 共识层客户端 |

## 部署建议

### 硬件要求（以太坊主网全节点）

- **CPU**：4 核以上
- **内存**：16GB 以上
- **存储**：2TB+ NVMe SSD
- **带宽**：25Mbps 以上

### 运维要点

1. 监控节点同步状态
2. 定期更新客户端版本
3. 磁盘空间监控和清理
4. 设置告警和自动重启
5. 使用反向代理和负载均衡

## 何时自建

- 需要完整归档数据
- 对隐私有严格要求
- 大量请求超出第三方额度
- 需要自定义 RPC 方法
